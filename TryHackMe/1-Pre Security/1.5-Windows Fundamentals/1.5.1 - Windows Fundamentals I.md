## 🖥️ The Desktop (GUI) – Windows 10 Arayüzü

Windows’a giriş yaptıktan sonra karşımıza çıkan grafiksel ekran "Desktop" yani masaüstüdür. Her kullanıcı bu masaüstü üzerinden bilgisayarı kullanır.

---

### 🔐 Giriş (Login Screen)
- Kullanıcı adı ve şifre girilerek oturum açılır.
- Domain'e bağlı sistemlerde Active Directory hesabı ile giriş yapılabilir.

---

### 🧱 Masaüstü (Desktop)
- **Kısayollar**, **klasörler**, **dosyalar** burada yer alır.
- Sağ tıklayarak:
  - İkon boyutu değiştirilebilir
  - Yeni klasör veya belge oluşturulabilir
  - Sıralama ve görünüm ayarları yapılabilir
- **Personalize** ile:
  - Arka plan (duvar kağıdı)
  - Renk temaları, yazı tipi vs. değiştirilebilir
- **Display Settings**:
  - Ekran çözünürlüğü, ekran yönü, çoklu ekran ayarları

📌 *Not:* Uzak masaüstü (RDP) oturumlarında bazı görüntü ayarları kısıtlanabilir.

---

### 🪟 Başlat Menüsü (Start Menu)
Windows logosuna tıklanarak açılır. 3 ana bölümden oluşur:

#### 1️⃣ Sol alt ikonlar:
- 🔒 Oturumu kilitle / çıkış yap
- ⚙️ Ayarlar (Settings)
- 🔌 Kapatma / Yeniden başlatma seçenekleri
- 📁 Belgeler / Resimler kısayolları

#### 2️⃣ Uygulama listesi:
- En son yüklenen uygulamalar en üstte
- Diğer tüm uygulamalar alfabetik sıralanır
- Harf üstüne tıklayarak harf listesine hızlı geçiş yapılabilir

#### 3️⃣ Sağ taraftaki **Tile** alanı:
- Sık kullanılan uygulamaların büyük ikonları
- Sağ tıklayıp: Boyutlandır, kaldır, başlat menüsüne sabitle gibi işlemler yapılabilir

---

### 🔳 Görev Çubuğu (Taskbar)
Ekranın altında yer alır. İçeriği:
- Açık olan uygulamalar burada görünür
- İkon üzerine gelince **önizleme ve tooltip** çıkar
- Sağ tıklayarak çubuğa uygulama sabitlenebilir

🛠️ Sağ tıklayıp **Toolbars**, **Taskbar ayarları** yapılabilir

---

### 🔔 Bildirim Alanı (Notification Area)
Genellikle sağ alt köşededir.

İçeriği:
- 🕓 Saat/tarih
- 🔊 Ses
- 🌐 Ağ durumu
- 💻 Donanım / yazılım uyarıları

📌 Ayarlardan simgeler gösterilip gizlenebilir:
```
Settings > Taskbar > Notification area
```

---

### 📌 İpucu:
**Her dosya, klasör veya uygulamaya sağ tıklayarak** detaylı bilgi veya ek seçenekler (özellikler, pinleme vs.) görülebilir.

---

## 💾 The File System (NTFS – Windows Dosya Sistemi)

---

### 📁 NTFS (New Technology File System)

Windows’ta kullanılan modern dosya sistemi. Önceki sistemlerden çok daha güvenli ve gelişmiştir.

#### 📌 Eski dosya sistemleri:
- **FAT16 / FAT32** → USB, SD kart gibi cihazlarda hâlâ kullanılır.
- **HPFS (High Performance File System)** → Artık pek kullanılmaz.

---

### ✅ NTFS Özellikleri
| Özellik                           | Açıklama                                                               |
|----------------------------------|------------------------------------------------------------------------|
| 📌 **Journaling**                | Diskteki değişiklikler loglanır. Sistem hatalarında otomatik onarım yapılabilir. |
| 📁 **4GB+ dosya desteği**        | FAT32’den farklı olarak çok büyük dosyalar saklanabilir.               |
| 🔐 **İzin ayarları (ACL)**       | Dosya ve klasörler için kullanıcı bazlı erişim kısıtlaması yapılabilir.|
| 🗜️ **Sıkıştırma (Compression)** | Dosya/klasör boyutları sıkıştırılabilir.                              |
| 🧾 **Şifreleme (EFS)**           | Dosya şifreleme özelliği (EFS – Encrypting File System)               |
| 🕵️ **ADS (Alternate Data Streams)** | Dosyanın arkasına gizli veri eklenebilir (güvenlik açığı olarak da kullanılır). |

---

### 🔒 NTFS İzinleri (Permissions)

Dosya/klasörlere sağ tıklayıp **Properties > Security** sekmesinden görüntülenir.

#### İzin Türleri:
- **Full Control** → Tüm işlemleri yapabilir
- **Modify** → Okuma/yazma/silme
- **Read & Execute** → Okuyabilir ve çalıştırabilir
- **List Folder Contents** → Klasör içeriğini listeleyebilir
- **Read** → Sadece okuyabilir
- **Write** → Sadece yazabilir

📌 *Kullanıcı veya grup seçilerek hangi izne sahip olduğu kontrol edilir.*

---

### 🧨 ADS (Alternate Data Streams)
- NTFS dosyalarının **arka planına veri gömmeye** olanak tanır.
- Windows Explorer bu verileri göstermez.
- **PowerShell** veya 3. parti araçlarla tespit edilebilir.
- Malware yazarları verileri bu yolla gizleyebilir.
- Ancak kötü amaçlı olmayan kullanımları da vardır:
  - Örn: İnternetten indirilen dosyalara kaynak bilgisi eklenir.

#### 🔎 Örnek Komut (PowerShell):
```powershell
Get-Item -Path .\dosya.txt -Stream *
```

📚 Daha fazla: Malwarebytes’in [ADS makalesi](https://www.malwarebytes.com/blog/news/2015/07/alternate-data-streams)

---

### 📍 C Sürücüsünün (C:\) NTFS olup olmadığını nasıl anlarsın?
- **Sağ tıkla > Properties > File system** yazan yere bak.
- Genelde “NTFS” olarak yazar.

---

### 🧠 Özet Tablo

| Konu         | Açıklama                                                         |
|--------------|------------------------------------------------------------------|
| NTFS         | Windows’ta kullanılan modern dosya sistemi                       |
| ADS          | Dosya arkasına görünmeyen veri ekleme (güvenlik için dikkat!)    |
| Permissions  | Dosya/klasör erişim izni yönetimi (Full Control, Read, Write...) |
| FAT/FAT32    | USB belleklerde yaygın, eski sistem                              |

## 🗂️ The Windows\System32 Folder

---

### 🪟 `C:\Windows` Klasörü

Windows işletim sisteminin **ana klasörüdür**. Genellikle `C:\Windows` dizininde bulunur, ancak farklı bir sürücüde veya klasörde de olabilir.

#### 🧠 Ortam Değişkeni (%windir%)
- Windows klasörü nerede olursa olsun, `%windir%` ortam değişkeni bu dizini gösterir.
- Komut satırında veya scriptlerde bu şekilde kullanılır:
  ```cmd
  cd %windir%
  ```

#### 📘 Ortam Değişkenleri (Environment Variables) nedir?
- Sistem hakkında bilgiler saklar:
  - İşletim sistemi yolu
  - İşlemci sayısı
  - Geçici klasör konumları
- Kullanım örneği:
  ```cmd
  echo %TEMP%
  echo %NUMBER_OF_PROCESSORS%
  ```

---

### 🔧 System32 Klasörü (`C:\Windows\System32`)

Windows klasörü içindeki **en kritik alt klasörlerden biridir**.

#### İçeriği:
- Windows’un **çekirdek sistem dosyaları** burada bulunur.
- .exe, .dll, .sys gibi **sistem bileşenleri** içerir.
- Aşağıdaki önemli araçlar genellikle buradadır:
  - `cmd.exe`
  - `taskmgr.exe`
  - `regedit.exe`
  - `ipconfig.exe`
  - `services.msc`

#### ⚠️ Uyarı:
- **Silme, taşıma veya değiştirme** işlemlerinde **çok dikkatli olunmalıdır**.
- Bu klasörde yapılan yanlış bir işlem, **Windows’un çökmesine** sebep olabilir!
- Sistem, bazı kritik dosyaları silmeyi engeller; ancak bu her zaman %100 koruma sağlamaz.

---

### 🧠 Özet Tablo

| Klasör               | Açıklama                                                      |
|----------------------|---------------------------------------------------------------|
| `%windir%`           | Windows klasörünü tanımlar (örn: `C:\Windows`)               |
| `%SystemRoot%`       | `%windir%` ile aynıdır, genellikle `C:\Windows`              |
| `System32`           | Sistem için **gerekli dosyaları** içerir                      |
| Ortam Değişkenleri   | Sistem bilgilerini ve yollarını tanımlar                      |

---

### 🎯 Bonus: Neden Bilmeliyim?

- **CMD / PowerShell komutları** çoğunlukla buradaki exe dosyalarıdır.
- Bir uygulamanın neden çalışmadığını araştırırken **System32 dosyaları eksik mi?** diye kontrol etmek gerekebilir.
- `%windir%\System32` yerine doğrudan "System32" yazabilmek için bu klasör ortam değişkeni PATH'e eklenmiştir.

## 👥 User Accounts, Profiles, and Permissions

---

### 👤 Kullanıcı Türleri

Windows sistemlerinde kullanıcılar 2 ana türdür:

| Kullanıcı Türü     | Yetkiler                                                                 |
|--------------------|--------------------------------------------------------------------------|
| **Administrator**  | Sistem ayarlarını değiştirme, kullanıcı ekleme/silme, yazılım kurma vb. |
| **Standard User**  | Sadece kendi klasörlerinde işlem yapabilir, sistem ayarlarını değiştiremez |

---

### 🔍 Kullanıcıları Görüntüleme (Grafik Arayüzle)

1. **Başlat Menüsü** → "Other User" yaz → **Other Users Ayarı**'nı aç.
2. Buradan:
   - Var olan kullanıcıları görebilirsin.
   - Yeni kullanıcı ekleyebilirsin (Admin yetkisiyle).
   - Mevcut hesabın türünü görebilir/değiştirebilirsin.

---

### 🗂️ Kullanıcı Profilleri (`C:\Users`)

Yeni bir kullanıcı hesabı oluşturulunca:

- Oturum açtığında **kullanıcı profili otomatik oluşur**.
- Profil yolu: `C:\Users\<kullanıcıadı>`
- Her kullanıcı profilinde varsayılan klasörler bulunur:
  - `Desktop`, `Documents`, `Downloads`, `Music`, `Pictures`

🛠 **Kullanıcı profili ilk oturumda User Profile Service tarafından oluşturulur.**

---

### ⚙️ Yerel Kullanıcı ve Grup Yönetimi (`lusrmgr.msc`)

- Hızlı erişim: `Win + R` → `lusrmgr.msc` yaz → Enter
- İki ana klasör göreceksin:
  - **Users**: Sistem üzerindeki tüm kullanıcılar
  - **Groups**: Yetkisel gruplar (örnek: Administrators, Users, Guests)

📌 Bir kullanıcıyı bir gruba eklersen, **o grubun yetkilerini otomatik alır.**
📌 Bir kullanıcı **birden fazla gruba** ait olabilir.

---

### 🧠 Özet Tablo

| Komut / Yol             | Açıklama                                                   |
|-------------------------|------------------------------------------------------------|
| `C:\Users\`             | Kullanıcı profillerinin bulunduğu klasör                   |
| `lusrmgr.msc`           | Yerel kullanıcı ve grup yönetimi aracı                     |
| `Administrator`         | Tam yetkili kullanıcı                                       |
| `Standard User`         | Sınırlı haklara sahip kullanıcı                             |
| "Add someone else..."   | Yeni kullanıcı ekleme seçeneği (sadece Adminlerde görünür)  |

---

### 🧪 Bonus Bilgi

✅ **Standard User**, yazılım kuramaz ve sistem dosyalarını değiştiremez.  
✅ **Admin kullanıcıları**, diğer tüm kullanıcıları ve grupları yönetebilir.

## 🛡️  User Account Control (UAC)

---

### ❓ UAC Nedir?

🔐 **User Account Control (UAC)**, yönetici haklarına sahip kullanıcıların sistemdeki **her işlemi doğrudan yönetici olarak çalıştırmasını engellemek** için geliştirilmiş bir güvenlik özelliğidir.

> 📌 **Amaç:** Zararlı yazılımların, kullanıcı farkında olmadan sistem üzerinde değişiklik yapmasını önlemek.

---

### 👤 UAC Nasıl Çalışır?

- Admin hesabıyla giriş yaptığında, işlemler **varsayılan olarak normal kullanıcı yetkisiyle** çalışır.
- Eğer bir işlem **yönetici yetkisi** gerektiriyorsa, UAC devreye girer ve kullanıcıdan **onay ister**.

🧭 Örnek UAC Süreci:

1. 🧑‍💻 Kullanıcı bir yazılım kurmaya çalışır.
2. 🛑 Windows, işlem için yönetici izni gerektiğini algılar.
3. 🧷 **UAC penceresi açılır.**
   - Admin hesabı için parola sorulur.
   - Onay verilmezse işlem gerçekleşmez.

---

### ⚠️ Notlar

- 📍 **UAC, yerleşik "Administrator" hesabı için varsayılan olarak devre dışıdır.**
- 🛡️ UAC etkinse, program simgelerinde küçük bir **kalkan (shield)** simgesi olur. Bu, işlemin **yükseltilmiş yetki** gerektirdiğini gösterir.
- 🧪 UAC sayesinde, bir programın sisteme zarar verme olasılığı düşer çünkü yetkisiz değişiklik yapılamaz.

---

### 🖥️ Standart Kullanıcı İle Kurulum Denemesi

- Standart kullanıcı oturumundayken kurulum yapmak istersen:
  - Program simgesinde **kalkan simgesi** çıkar.
  - Tıklayınca, UAC açılır ve **Admin parolası** ister.
  - Parola girilmezse kurulum **gerçekleşmez**.

---

### 🧠 Kısa Bilgiler Tablosu

| Özellik                    | Açıklama                                                                 |
|----------------------------|--------------------------------------------------------------------------|
| UAC Nedir?                 | Yükseltilmiş işlemler için onay isteyen güvenlik katmanıdır              |
| Kimler etkilenir?         | Admin hesabı dışındaki tüm kullanıcılar                                 |
| UAC ne yapar?              | Yetki isteyen işlemleri durdurur, kullanıcıdan onay/parola ister         |
| Shield simgesi ne demek? | UAC onayı gerektiren işlem demektir                                     |
| Standart kullanıcı etkisi | Program kurmak için Admin onayı gerekir                                 |

## 🧠 Task Manager (Görev Yöneticisi)

---

### 📋 Nedir?

Task Manager, sistemde çalışan uygulamaları ve işlemleri **canlı olarak izlememizi** sağlayan bir araçtır. Sistem kaynaklarının kullanımı hakkında da bilgi verir (CPU, RAM, Disk, Ağ vs.).

---

### 🧭 Nasıl Açılır?

🖱️ 1. Görev çubuğuna sağ tıkla → `Task Manager (Görev Yöneticisi)`  
⌨️ 2. `Ctrl + Shift + Esc`  
⌨️ 3. `Ctrl + Alt + Del → Task Manager`

---

### 🧊 Basit Görünüm (Simple View)

İlk açılışta yalnızca **açık uygulamalar** görünür.
🔽 Devamını görmek için:  
**More Details** butonuna tıklayın.

---

### 🧩 Geniş Görünümde (More Details) Sekmeler:

| Sekme            | Açıklama                                                                 |
|------------------|--------------------------------------------------------------------------|
| 🔄 **Processes**  | Aktif uygulamalar & işlemler, CPU/RAM/Disk/Ağ kullanımı                  |
| ⚙️ **Details**     | Tüm işlemlerin detaylı listesi, işlem ID’leriyle birlikte                |
| 📊 **Performance**| Sistem kaynakları (CPU, RAM, Disk, Ethernet) grafikli şekilde           |
| 🚀 **Startup**     | Başlangıçta otomatik açılan uygulamaların listesi                       |
| 👥 **Users**        | O anda sisteme giriş yapmış kullanıcılar ve kaynak kullanımları         |
| 🔌 **Services**     | Windows servisleri (çalışan/duran) ve açıklamaları                      |
| 🌐 **App history** | (Bazı sistemlerde) Uygulamaların geçmiş kaynak kullanımı                |

---

### 📈 Performance Sekmesi

Bu sekme, sistem kaynaklarının anlık kullanımını **grafikli** olarak gösterir.

🧮 Görülebilen bilgiler:

- CPU kullanım yüzdesi
- Bellek (RAM) kullanımı
- Disk okuma/yazma
- Ağ (Ethernet/Wi-Fi) trafiği
- GPU kullanımı

---

### 🧼 Ne İşe Yarar?

- Yanıt vermeyen programları kapatma
- Sistemi yoran işlemleri belirleme
- Açılışta otomatik çalışan uygulamaları devre dışı bırakma
- Servisleri başlatma/durdurma

---

📚 Windows’un çekirdek işlemleri hakkında daha fazla bilgi edinmek istersen:  
**Core Windows Processes** bölümüne göz atabilirsin.
