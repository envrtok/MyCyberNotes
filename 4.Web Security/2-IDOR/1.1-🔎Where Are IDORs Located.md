### 1. Not Always in the Address Bar

- Vulnerable endpoints aren’t always visible in the URL.
- They may be loaded dynamically through **AJAX requests** or referenced inside **JavaScript files**.

### 2. Hidden or Legacy Parameters

- Some endpoints include parameters left over from development that still exist in production.
- These parameters may not be documented or directly referenced in the UI.

### 3. Example: Parameter Mining

- Normal request:
    
    ```
    /user/details
    ```
    
    → Displays your own user information (via session authentication).
    
- Discovered parameter:
    
    ```
    /user/details?user_id=123
    ```
    
    → By manipulating `user_id`, you can access other users’ data.
